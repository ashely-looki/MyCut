# MyCut 部署说明（Docker，单机）

把整套（前端 + 后端 + FFmpeg + Remotion 成片）打成两个容器，一条命令起。
适合「一台云服务器让别人打开网页就能用」的场景。

## 架构

```
浏览器 ──▶ frontend 容器 (nginx:80)
                 ├── 托管 vite 打包的静态前端
                 └── 反代 /api、/health、/api/v1/ws(WebSocket) ──▶ backend 容器 (uvicorn:8000)
                                                                        ├── FastAPI
                                                                        ├── FFmpeg（切片/抽色）
                                                                        ├── Node + Remotion + headless Chromium（成片）
                                                                        └── SQLite + 本地文件产物（挂持久卷 /app/data）
```

- **任务执行**：后端容器用「桌面语义后台线程」跑切片/成片（`AUTOCLIP_DESKTOP_MODE=1`），**无需 Redis / Celery worker**，单容器即可。
- **数据持久化**：SQLite、上传、切片、成片都在 `/app/data`（docker volume `mycut_data`），容器重建不丢。

## 一、服务器要求

- 装了 Docker + Docker Compose（`docker compose version` 能跑）。
- **内存建议 ≥ 4GB**：Remotion 用 headless Chromium 逐帧渲染 **1080p** 成片，峰值内存 1~1.5G。2G 机器成片易 OOM；**2核4G 是能稳定出片的下限**（需加 swap + 限并发，见下）；4核8G 更舒适、渲染也更快。只做「上传→切片」的话 2G 够。
- 能出公网（LLM API、edge-tts 在线配音、Higgsfield 都要联网）。
- 磁盘 **≥ 50GB**：镜像约 3G，且成片 mp4 与实拍缓存**会永久累积**（单容器不跑自动清理），需定期跑 `deploy/cleanup-data.sh`。
- 镜像较大（后端约 2.8GB，含 Chromium + 中文字体），首次构建下载较久。

## 一·A、阿里云海外（新加坡/香港，免备案）2核4G 快速部署 ★

> 这是针对「阿里云海外 2核4G、IP 直连、免备案」的一键路径。国内机房见文末补充。

**为什么选海外**：阿里云海外机房（新加坡/香港等）**无需 ICP 备案**，80/443 端口可直接对公网服务、用 IP 访问。大陆用户为主选**香港**（延迟最低），其次新加坡。

**买机器时选**：2核4G、系统盘 ≥50G、镜像选 Ubuntu 22.04 或 Alibaba Cloud Linux 3、按量带宽或固定带宽按预算。

**部署三步**：

```bash
# 1. 拉代码 + 准备配置
git clone <你的仓库> && cd my-clip-agent
cp .env.production.example .env.production
vi .env.production          # 确认 REMOTION_CONCURRENCY=2；要认证/支付再填对应块

# 2. 一键部署（加 swap 防 OOM + 装 Docker + 构建起服务）
sudo bash deploy/aliyun-setup.sh

# 3. 阿里云控制台 → 该实例「安全组」→ 放行入方向 TCP 80 端口（0.0.0.0/0）
#    ★ 这一步最容易漏！不放行则网页打不开（默认只开 22）。
```

访问 `http://<公网IP>/`。健康检查 `curl http://<公网IP>/health`。

**2核4G 专属注意**：
- 脚本已自动加 **2G swap** 并把 `vm.swappiness=10`（成片渲染 OOM 兜底）。
- `.env.production` 里 **`REMOTION_CONCURRENCY=2`**（限制 Chromium 实例数，防撑爆内存）。别调大，除非升配到 4核8G 可设 3~4。
- 成片速度：2核会打满 CPU，一条几十秒成片可能渲染数分钟，属正常。
- **磁盘会涨**：挂个定时清理（每天清 30 天前产物）：
  ```bash
  crontab -e   # 加一行：
  0 3 * * * cd /root/my-clip-agent && bash deploy/cleanup-data.sh 30 >> /var/log/mycut-cleanup.log 2>&1
  ```

## 二、部署步骤（通用）

```bash
# 1. 拉代码到服务器
git clone <你的仓库> && cd my-clip-agent

# 2. 准备生产配置
cp .env.production.example .env.production
#    编辑 .env.production：至少确认 ENVIRONMENT/DEBUG；要认证/支付再填对应块。

# 3.（可选）接支付宝时，把两把 PEM 放到 ./secrets/
#    mkdir -p secrets && cp 你的私钥.pem secrets/alipay_app_private_key.pem ...

# 4. 构建并启动
docker compose up -d --build

# 5. 访问
#    http://<服务器IP>/     （前端 80 端口）
#    健康检查：curl http://<服务器IP>/health
```

停止 / 更新：

```bash
docker compose down           # 停止（数据卷保留）
git pull && docker compose up -d --build   # 更新代码后重建
```

## 三、关键配置说明

| 项 | 同源部署（推荐，默认） | 前后端分离部署 |
|---|---|---|
| 前端调后端 | 相对路径 `/api/v1`，nginx 反代，无需配 | 前端构建时设 `VITE_API_BASE_URL=https://api域名`（改 Dockerfile.frontend 的 build ARG） |
| CORS | `CORS_ALLOW_ORIGINS` 留空即可 | 填后端要放行的前端域名（逗号分隔） |
| WebSocket 进度 | 自动按当前域名推导 `wss://域名/api/v1/ws` | 由 `VITE_API_BASE_URL` 推导 |

- **HTTPS**：compose 里前端是 80。要 HTTPS，最简单是在前面再套一层带证书的反向代理（Caddy / nginx / 云厂商负载均衡），或用 Cloudflare 套 CDN+证书。支付宝 notify_url、生产环境都建议上 HTTPS。

## 四、成片功能的现实

- **配音（edge-tts）/ 组件 / 字幕 / Remotion 渲染**：容器内齐全，能出成片。
- **实拍空镜（Higgsfield）**：需 OAuth 登录 + 付费额度，容器内默认未登录 → **自动降级为信息动画**，成片仍成功产出。要在服务器出实拍空镜，需自行把 Higgsfield 登录态/凭证配到后端容器（另行处理）。

## 五、常见问题

- **成片字幕是方块**：镜像已装 `fonts-noto-cjk`；若自行改了基础镜像需确认中文字体在。
- **成片 OOM / 渲染超时**：先确认加了 swap（`swapon --show`）且 `REMOTION_CONCURRENCY=2`（小内存别调大）；仍不行就升内存，或临时只用「上传→切片」。
- **上传大视频 413**：nginx 已放开到 2048m（见 `deploy/nginx.conf` `client_max_body_size`），需要更大自行调。
- **任务卡住不动**：确认后端容器 `AUTOCLIP_DESKTOP_MODE=1`（compose 已设）；看 `docker compose logs backend`。
- **磁盘满**：跑 `bash deploy/cleanup-data.sh 30`（清 30 天前成片/实拍缓存），或挂 crontab 定期清。
- **想抗并发/多副本**：当前单容器 + SQLite 不适合横向扩展。届时切 Postgres（`DATABASE_URL`）并拆 API + Celery worker + Redis（代码已具备 web 模式）。

## 六、国内机房构建（备用）

若将来换成**国内机房**服务器，直连 Docker Hub / npm / Chromium 官方源会很慢甚至超时。构建时加国内源开关：

```bash
docker compose build --build-arg CN_MIRROR=1   # apt→阿里云、pip→阿里云、npm→npmmirror
docker compose up -d
```

海外机器**不要**加 `CN_MIRROR=1`（走国内源反而更慢）。
