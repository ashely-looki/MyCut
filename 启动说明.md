# my-clip-agent 启动说明 & 阶段进度

> 这是从 `autoclip-main` 拷贝改造而来的自有工程。整体规划见上级目录 `../架构方案.md`。

## 当前进度：阶段 0 ✅ 真正闭环（已出真切片）

上传本地视频 → step1~6 全链路跑通 → **产出带标题的精彩切片**。已用 DeepSeek 实测：
90s 视频 → 14s 跑完 → 2 个切片（评分 0.95 / 0.93），mp4 落盘、项目 completed。

**LLM 用的是 DeepSeek**（key 存 `data/settings.json`，已 gitignore，不入库）：
- provider = `deepseek`，模型 `deepseek-chat`（V3）。切换模型可改 settings.json 的 `model_name`
  为 `deepseek-reasoner`（R1）。
- DeepSeek 走 OpenAI 兼容接口（base_url=https://api.deepseek.com），代码在
  `backend/core/llm_providers.py` 的 `DeepSeekProvider`。

### 已删范围（本项目只做本地上传切片）
B站上传/投稿/下载、YouTube 下载、桌面壳、Docker、投稿队列、账号管理全部移除。
入口只剩「本地文件上传」。

### 阶段 0 修复的关键 bug（autoclip 原代码）
1. `projects.py` upload 里未定义的 `db` → 改 `project_service.db`。
2. 桌面模式 `update_state()` 连 Redis → `DesktopAwareTask` 重写为日志。
3. **SQLite `StaticPool`（全进程单连接）在多线程任务下事务污染** → 报
   `cannot commit - no transaction is active`，流水线崩溃。改用默认 QueuePool
   （每线程独立连接）+ `check_same_thread=False`。（`backend/core/database.py`）

### 阶段 0 修复的两个阻断 bug（autoclip 原代码就有）
1. **`backend/api/v1/projects.py`** — `upload_files` 里用了未定义的 `db`，导致上传后异步任务启动即崩。
   已改为 `project_service.db`。
2. **`backend/core/celery_app.py`** — 桌面模式下任务调 `self.update_state()` 仍连 Redis(6379)，
   没 Redis 直接失败。已在 `DesktopAwareTask` 重写 `update_state`，桌面模式降级为日志。

### 新增文件
- `backend/desktop_start.py` — 桌面模式启动入口（先加载 .env、锁定桌面模式，再以 desktop 模式建 app）。
- `.env` — 本地配置（桌面模式、数据目录、FFmpeg 路径）。
