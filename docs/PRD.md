# Vibe Coding PRD：MyCut — AI 视频剪辑 Agent

> 面试用版本。目标：讲清产品闭环、技术架构与关键工程决策，并给出可验证的验收标准。
> 本文件为独立文档，不替代 `DESIGN.md`（设计系统）与 `README.md`。

## 1. 需求定义

- **用户**：内容创作者 / 短视频剪辑者（个人创作者为主）。
- **场景**：从"不知道拍什么"到"出成片"的完整创作过程——选题、写稿、拍摄后处理、成片，桌面端本地完成。
- **痛点**：
  - 选题难：不知道当下什么话题有热度。
  - 写稿慢：从选题到可拍的分镜文案要反复打磨。
  - 剪辑苦：长视频里挑精彩片段、配字幕、起标题耗时。
  - 出片重：想做"实拍 + 动态信息组件"的口播/科普视频，普通人没有能力手工做转场、动效、配音、字幕对齐。
- **产品形态**：macOS 桌面应用（前后端分离，本地运行，无需 Redis）。
- **本次版本目标**：完整创作闭环已跑通（查热点 → 文案 → 上传切片 → 自动成片）；本 PRD 覆盖全产品，并把"自动成片（实拍 + 组件叠加）"作为核心亮点模块。
- **核心输入**：领域/关键词、拍摄的本地视频（可选 SRT）、保存的文案。
- **核心输出**：选题卡片、结构化大纲与分镜文案、精彩切片 + 标题 + 合集、一键生成的成片 MP4（实拍空镜 + 动态组件 + 配音 + 字幕）。
- **关键限制**：桌面本地运行；LLM 用 DeepSeek；不含任何平台下载/投稿；密钥存本地 `data/settings.json`（不入库）；实拍生成依赖 Higgsfield（付费额度），不可用时自动降级为纯信息动画。

## 3. 技术栈建议

> 说明：本项目已落地下述技术栈，此处记录选型与理由（面试可复述工程判断）。

- **推荐方案**：前后端分离 + 桌面本地运行，重任务后台线程化，AI 能力用 API/CLI 组合而非自建模型。
- **前端**：React 18 + TypeScript + Vite + Ant Design（现状）。设计规范见 `DESIGN.md`（Calm Premium 克制专业风）。
- **后端/API**：Python + FastAPI；任务用桌面模式后台线程（`DesktopAwareTask`），**无需 Redis**，开箱即用。
- **数据库/存储**：SQLite（QueuePool + `check_same_thread=False`，规避多线程事务污染）；产物落本地文件。
- **AI / Agent / Tool**：
  - LLM：DeepSeek（OpenAI 兼容接口，`deepseek-chat`）——查热点、大纲、文案、视觉脚本、视频 prompt。
  - 配音：edge-tts（免费免 key，中文自然）。
  - 实拍视频：Higgsfield CLI（Kling/Veo/Seedance 等模型，OAuth 本地登录）。
  - 合成：Remotion（React 写视频，逐帧渲染 + ffmpeg 出 MP4）。
  - 切割/探测：FFmpeg / ffprobe。
- **部署**：macOS 桌面本地；后端 `python -m backend.desktop_start --port 8000`，前端 Vite `:3000`，`/api` 代理到后端。
- **推荐理由**：非技术用户要"开箱即用"→ 去掉 Redis、用 SQLite；AI 能力"组合现成服务"→ 不自建模型、迭代快、成本可控；成片用 Remotion 而非手工剪辑 → 数据驱动、可批量、可复现。
- **功能与技术对应**：查热点/文案/视觉脚本=DeepSeek；配音=edge-tts；实拍=Higgsfield；合成=Remotion+FFmpeg；切片=FFmpeg+LLM 评分；持久化=SQLite+本地文件。

## 4. 核心界面原型（框架版）

### 4.1 首页（创作入口）
- **页面目标**：一眼看到三条入口——查热点、文案库、上传切片。
- **核心区块**：查热点入口卡、自动成片入口卡（选文案→生成视频）、最近项目列表。
- **关键控件**：领域/关键词输入框、"查热点"按钮、项目卡片（状态点 + 进度）。
- **主要交互**：输入领域→查热点→选题→写文案；或直接进文案库/上传。
- **状态反馈**：项目卡左上角状态点（下载中/分析中=accent、已完成=ok、失败=error）+ 进度细线。

### 4.2 文案库
- **页面目标**：管理已存文案，一键进入剪辑或成片。
- **核心区块**：文案卡网格（标题、开场白、段数、时长、类型）。
- **关键控件**：每卡「编辑 / 用它剪视频 / 生成视频 / 删除」。
- **主要交互**：点「生成视频」→ 后端自动成片 → 首页看进度 → 详情页看成片。
- **状态反馈**：生成失败顶部提示失败原因，不清空列表。

### 4.3 文案编辑
- **页面目标**：编辑大纲与逐镜文案。
- **核心区块**：大纲区（hook/分段/CTA）、逐段文案编辑区。
- **关键控件**：段落编辑、保存、生成视频。
- **状态反馈**：未生成文案时提示"请先生成文案再生成视频"。

### 4.4 项目详情（切片 / 成片产物）
- **页面目标**：查看切片和成片 MP4，播放/下载。
- **核心区块**：切片列表（ClipCard）、成片播放器。
- **关键控件**：播放、下载视频。
- **状态反馈**：完成元信息用纯灰 mono（例 `7 切片 · 1 合集`）。

## 5. 明细功能设计

### 5.1 AI 查热点
- **目标**：把"不知道拍什么"变成可选选题卡。
- **入口**：首页输入领域/关键词 → 查热点。
- **输入**：领域/关键词。
- **处理逻辑**：热点服务归纳（数据源可插拔，默认 LLM 兜底，可接 Bing 抓取）→ 产出选题卡片。
- **输出**：选题卡片列表（标题 + 推荐理由）。
- **异常/边界**：数据源失败回退纯 LLM 生成；无结果给空态提示。

### 5.2 大纲 + 分镜文案
- **目标**：从选题产出可拍的结构化脚本。
- **入口**：选题卡 → 生成大纲 → 生成分镜文案。
- **输入**：选定选题。
- **处理逻辑**：LLM 生成 hook/分段/CTA 大纲 → 逐镜文案 → 可就地编辑 → 保存进文案库。
- **输出**：结构化文案（title / segments[narration, role] / style）。
- **异常/边界**：LLM 失败可重试；空文案不允许进入成片。

### 5.3 本地上传 + 六步切片流水线
- **目标**：把长视频自动切成带标题的精彩片段。
- **入口**：上传视频（可选 SRT，可关联文案）。
- **处理逻辑**：step1 大纲 → step2 时间区间 → step3 评分（有文案时 step0 抽要点、评分偏向匹配脚本）→ step4 标题 → step5 聚类 → step6 FFmpeg 切割。
- **输出**：切片 MP4 + 标题 + 合集。
- **异常/边界**：缩略图对纯测试视频可能失败，不影响主流程；任务失败置 project=FAILED 并记录原因。

### 5.4 自动成片（核心亮点：实拍 + 组件叠加）
- **目标**：一篇文案 → 一条"实拍空镜 + 动态信息组件 + 配音 + 字幕"的成片，用户零额外操作。
- **入口**：文案库/编辑页「生成视频」→ `POST /compose/from-script`。
- **输入**：文案（title / segments）。
- **处理逻辑**（`compose_service.build_props`，分三阶段）：
  1. **配音（串行快）**：每句 edge-tts 合成，ffprobe 测真实时长，排字幕时间轴。
  2. **实拍生成（并发）**：每句 → `video_prompt_service` 出英文空镜 prompt（尽量每句都出，抽象句用隐喻空镜；强制无人脸/无文字）→ `higgsfield_service` 线程池并发生成（默认 4 并发）→ 下载落 `remotion/public/`。含估价、额度上限保护、持久缓存（按句子原文 hash，同句不重生）、降级（失败/不可用回退信息动画）。
  3. **组件与主题（每句）**：`scene_service` 生成视觉脚本（关键词/图标/步骤/对比），只提炼**概念词**且去重（不照抄原句，避免与字幕重复）；组件入场时机按关键词在原句的**字符位置比例**对齐配音；实拍句抽视频主色生成呼应画面的 `overlayTheme`。
- **输出**：Remotion `inputProps` → `remotion render` 出 `compose.mp4`（1920×1080）。前端：实拍全屏铺底 + 组件弹性叠加（overlay 模式，实底卡片带阴影）+ 底部白字字幕（硬切轨、不重影）+ 句间 fade 溶解。
- **异常/边界**：Higgsfield 未登录/不可用 → 整条回退信息动画，不中断；下载优先 curl（规避 macOS Python SSL 证书问题）；渲染 headless 浏览器超时放宽到 120s；额度达上限后续句回退。

## 6. AI Agent 设计

### 6.1 成片 Agent 工作流
- **触发**：用户点「生成视频」（`/compose/from-script`），后台线程执行。
- **输入**：保存的文案 dict。
- **步骤**：定调色 → 逐句配音 → 逐句判断/生成视频 prompt → 并发生成实拍 → 逐句生成组件脚本 + 呼应色 → 组 props → Remotion 渲染。
- **Tool 调用**：DeepSeek（文案→视觉脚本/视频 prompt）、edge-tts（配音）、Higgsfield CLI（实拍）、FFmpeg（抽色/探测）、Remotion CLI（渲染）。
- **输出**：`compose.mp4`，登记为 Clip，项目置完成。
- **失败处理**：任一句实拍失败→该句回退信息动画；整体不可用→全回退；渲染失败→置 FAILED 记录原因。
- **人工确认点**：无（一键成片）；成本通过 `HIGGSFIELD_MAX_CREDITS` 上限自动保护。

### 6.2 提示词设计（关键两处）
- **视频提示词（`prompt/视频提示词.txt`）**：
  - System：纪实短视频「镜头导演」，每句都提炼可拍空镜（具象拍场景、抽象用隐喻）。
  - 输出格式：`{visual: bool, prompt: 英文40~80词, motion: 运镜}`。
  - 约束：必含 `no faces, no text, no watermark`；人只能背影/剪影/局部。
- **视觉脚本（`prompt/视觉脚本.txt`）**：
  - System：信息可视化导演，把一句话提炼成 1~3 个**概念词**（绝不照抄原句，避免与字幕重复）。
  - 输出格式：`{layout: keyword|steps|arrow|compare, elements:[{type,text,icon,enterAt,emphasis}]}`。
  - 约束：概念词≤3 个、图标限定词表、能在原句连续找到的词判为照抄并丢弃。

### 6.3 Tool 体系
| Tool | 用途 | 输入 | 输出 | 失败处理 |
|---|---|---|---|---|
| DeepSeek LLM | 热点/大纲/文案/视觉脚本/视频 prompt | prompt + JSON input | 结构化 JSON | 重试；解析失败降级兜底 |
| edge-tts | 逐句配音 | 文本 + 声音 | mp3 + 时长 | 失败用无配音兜底时长 |
| Higgsfield CLI | 生成实拍空镜 | 英文 prompt + 时长/比例 | result_url(mp4) | 返回 None，该句回退信息动画 |
| FFmpeg/ffprobe | 抽视频主色 / 探测时长 | 视频文件 | 主色 HEX / 秒 | 失败用统一 theme / 兜底时长 |
| Remotion CLI | 逐帧渲染合成 | props.json | compose.mp4 | 超时/失败置任务 FAILED |

## 7. 数据与权限

- **核心数据对象**：Project（项目/成片载体）、Script（文案）、Clip（切片/成片）、Collection（合集）、Task（任务状态）。
- **关键字段**：Script(title/segments[narration,role]/style)；Segment 渲染态(visualType/visualSrc/scene/overlayTheme/durationInFrames)；Project(status/video_path)。
- **权限角色**：单机单用户，无多角色权限。
- **数据保留/导入导出**：产物本地留存；测试成片与素材缓存不入库（.gitignore）；成片支持下载。

## 8. 验收标准

### 8.1 查热点 → 文案
- [ ] 输入领域并点「查热点」后，出现≥1 张选题卡片；数据源失败时回退 LLM 生成且不报错。
- [ ] 选定选题点生成后，产出含 hook/分段/CTA 的大纲，且可就地编辑并保存进文案库。

### 8.2 上传切片
- [ ] 上传视频后启动六步流水线，完成后详情页出现≥1 个切片 MP4 + 标题。
- [ ] 关联文案时，评分偏向匹配脚本要点的片段；无文案时为常规切片，无报错。

### 8.3 自动成片
- [ ] 文案库点「生成视频」返回成功并提示去首页看进度；失败时顶部显示原因、不清空列表。
- [ ] 成片为 1920×1080 MP4：实拍空镜全屏铺底 + 动态组件叠加 + 配音 + 底部字幕。
- [ ] 组件文字为概念词，不出现与底部字幕一字不差的连续重复。
- [ ] 组件入场时机跟随配音（句首词早出、句尾词晚出），不早于画面稳定、不超出音频。
- [ ] 组件强调色与该句实拍画面同色系（accent 取自视频主色）。
- [ ] 句间为 fade 溶解，相邻句字幕不同时半透明重叠。
- [ ] Higgsfield 未登录/不可用时，整条自动回退为信息动画且成片成功产出。
- [ ] 同一句文案重复成片时命中持久缓存，不重新消耗生成额度。
- [ ] 生成额度累计达 `HIGGSFIELD_MAX_CREDITS` 后，后续句回退信息动画而非继续消耗。

### 8.4 系统
- [ ] 后端桌面模式下任务在后台线程执行，无需 Redis 即可完成切片与成片。
- [ ] 密钥仅存本地 `data/settings.json`，不出现在仓库或渲染产物中。

## 9. 不做范围
- 任何平台的下载/投稿/账号管理（已明确移除）。
- 多用户/权限系统、云端部署。
- 词级精确字幕对齐（当前用字符位置近似；faster-whisper 精确对齐列为后续）。
- 前端"成片形态开关"（当前默认实拍 + 组件叠加，暂不暴露选项）。
- 真人正脸出镜 / AI 写实人物生成（Remotion 与空镜方案不覆盖）。

## 10. 给 Claude Code / Codex 的实现备注
- 先保证 MVP 核心闭环：查热点 → 文案 → 上传切片 → 自动成片，再做 P1/P2。
- 不实现"本期是否做=否"的功能（词级对齐、成片形态开关）。
- 成片是**旁路模块**：任何实拍/组件失败都必须能回退信息动画，绝不中断成片主流程。
- 涉及外部服务（Higgsfield/DeepSeek）一律"不可用即降级"，密钥走本地配置、不硬编码、不入库。
- 遇到 PRD 未说明的行为，选最小可用默认方案并用 TODO 标注；保持交互与验收标准一致。
